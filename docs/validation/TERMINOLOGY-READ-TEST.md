# Terminology Read Test

## Purpose
Check comprehension of result-screen terminology among people unfamiliar with the product.

This is a terminology check only, not product validation.

## Participant Target
- 5 to 8 participants
- No prior involvement with Work With Me design or implementation

## Test Material
Present participants with:
- Current heading/explanation from result context (including the term permanent instructions)
- Candidate alternative term: reusable baseline instructions
- Supporting sentence:
  - Use these instructions as long as they help. You can return and create a new set whenever your needs change.

## Core Open Question
Ask first, without answer options:

If your work situation changed significantly in six months, what would you do about these instructions?

## Additional Questions (max three)
1. What does permanent instructions mean to you here?
2. Does this feel like a description of you, or a tool setting you can change?
3. Which heading is clearer: permanent instructions or reusable baseline instructions, and why?

## Interview Procedure
- Ask core question first.
- Do not provide multiple-choice answers before participant response.
- Capture verbatim responses.
- Use same script for all participants.
- Do not explain intended product meaning until after all answers are recorded.

## Coding Guide
Classify each participant response into one primary category:
- understood as reusable and replaceable
- understood as permanently stored
- understood as identity-level or immutable
- unclear
- other

Recommended coding notes:
- include exact quote fragment
- include confidence (high/medium/low)
- include reasons for classification

## Output Template
Per participant record:
- participantId
- coreResponse
- q1Response
- q2Response
- q3Response
- primaryCategory
- coderNotes

Aggregate summary:
- count per category
- major misunderstandings
- candidate wording recommendation (if any)
