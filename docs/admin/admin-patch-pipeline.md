# Admin Patch Pipeline (V1.2 Foundation)

This workflow turns FW-ADMIN-001's "Export Patch" output into a repeatable pipeline edit. It applies the patch to the SOURCE category JSON files (`content/categories/*.json`), not the generated asset.

## Supported Patch Schema
Array of operations generated by `/admin/content`. Four operation types:

### Text Edit Operations
- `id`: string — category, position, or challenge id
- `kind`: `"category"` | `"position"` | `"challenge"`
- `field`: string — field name
  - category: `"name"` | `"description"`
  - position: `"statement"`
  - challenge: `"triggerRule"` | `"title"` | `"body"`
- `value`: string (category/position/challenge-string) or object (challenge-triggerRule)
  - category/position: new text value (trimmed)
  - challenge: triggerRule object `{parentAnswerMin?: number, parentAnswerMax?: number, tags?: string[]}` or title/body string (trimmed)
- Only safe edits allowed; unknown ids/fields skipped, non-allowlisted fields abort
- Challenge edits update nested challenges in position.deeperDives[] arrays

### Reorder Operations
#### Position Reorder
- `op`: `"reorder"`
- `kind`: `"position"`
- `categoryId`: string — target category id
- `orderedIds`: string[] — complete ordered list of position ids
- Must provide all position ids in desired order; unknown categoryId skipped

#### Category Reorder
- `op`: `"reorder"`
- `kind`: `"category"`
- `orderedIds`: string[] — complete ordered list of category ids
- Must provide all category ids in desired order; unknown ids skipped
- Updates the `order` field on each category file (first id in orderedIds gets `order: 0`, second gets `order: 1`, etc.)

### Set Hidden Operations
- `op`: `"setHidden"`
- `kind`: `"position"`
- `id`: string — position id
- `categoryId`: string — parent category id
- `value`: boolean — true to hide, false to unhide
- Unknown ids skipped with reason logged

## Workflow
1. Visit `/admin/content` and make the desired text edits.
2. Click **Export Patch** and save the generated JSON file (example: `patches/2025-12-23-liberty.json`).
3. Run a dry run (default mode):

        npm run admin:apply-patch -- --patch ./patches/2025-12-23-liberty.json

   - Validates the patch, shows applied/skipped operations, and leaves files untouched.
4. To persist edits, re-run with `--write`:

        npm run admin:apply-patch -- --patch ./patches/2025-12-23-liberty.json --write

   - Writes only the touched category files with stable JSON formatting (2 spaces + trailing newline).
5. Rebuild and export content when ready:
   - `npm run content:build`
   - `npm run content:export-app`

## Safety Notes
- Dry run is the default; `--write` must be explicit.
- Script stops if a patch references unsupported fields.
- Unknown ids stay untouched but are reported with reasons.
- After any write, include the regenerated artifacts in the same commit.
