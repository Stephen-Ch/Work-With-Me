# Test Catalog — Rawls Game

## Purpose
Central registry of all test specs. Before editing any `*.spec.ts`, read this catalog. After editing, update the corresponding row.

## How to Use
1. Find the spec you're about to touch
2. Read its "@human" summary and "What it proves"
3. After changes, update the row (or add a new one)
4. Include "Tests touched:" in your completion report

## Update Rule
If you touch a spec file, update its catalog row in the same prompt.

## Spec Header Standard
Every spec file should include a comment block at the top:
```typescript
/**
 * @human <one-sentence purpose>
 * @proves <what behavior this guards>
 * @lastTouched <YYYY-MM-DD>
 */
```

## Adoption Rule
If you touch a spec file and it lacks the Spec Header Standard, add it in that prompt.
